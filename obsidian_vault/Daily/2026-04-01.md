## Architecture Analysis: Voice Turn Detection & Routing

### Problem Statement
Two independent failure modes causing voice agent issues:
1. **Voice turns fragment before routing** — STT fires on partial utterances ("just") triggering incorrect research routes
2. **Router classifies in isolation** — AgentRouter sees only current utterance without conversation history
3. **Flutter doesn't detect backend death** — When Python agent dies, LiveKit room persists, Flutter shows "connected"

### Root Causes Identified

#### Turn Fragmentation
- Location: `agent.py:2009-2144` (`_on_user_input_transcribed`)
- VAD-based endpointing knows *when* speech stopped, not *whether* thought is complete
- Current EOU logic uses `min_endpointing_delay`/`max_endpointing_delay` + coalescer
- **Gap:** No semantic completeness check

#### Stateless Routing
- Location: `core/orchestrator/agent_router.py`
- `AgentRouter.route()` receives `utterance` and `user_id` but **NO conversation history**
- Architecture already has `chat_ctx` from LiveKit's AgentSession
- **Gap:** `chat_ctx` not passed to routing decision

#### Backend Liveness
- Location: Flutter `session_provider.dart`
- Only watches LiveKit room state, not agent process state
- **Gap:** No heartbeat from backend agent to Flutter client

### Recommended Fixes

| Priority | Fix | Impact | Risk |
|----------|-----|--------|------|
| 1 | Thread `chat_ctx` to AgentRouter | High | Low |
| 2 | Add utterance completeness guard | High | Low |
| 3 | Enable LiveKit EOU Model | High | Low |
| 4 | Add agent heartbeat (Flutter) | Medium | Low |

### Fix Details

#### Fix 1: Enable LiveKit EOU Model
```python
from livekit.plugins.turn_detector import EOUModel

session = AgentSession(
    vad=silero.VAD.load(),
    turn_detection=EOUModel(),  # Semantic turn detection
    min_endpointing_delay=1.0,
    max_endpointing_delay=6.0,
)
```
- 98.8% accuracy on completed turns
- 85% reduction in unintentional interruptions

#### Fix 2: Utterance Completeness Guard
Add pre-routing check in `agent_orchestrator.py`:
- Short utterances (<3 words) without verbs → route to CONVERSATION
- Continuation markers ("just", "and", "but") with prior context → hold for clarification
- Single-word commands ("yes", "stop", "play") → allow through

#### Fix 3: Context-Aware Routing
Update `AgentRouter.route()` signature:
```python
async def route(self, utterance: str, user_id: str, chat_ctx: list = None) -> str:
```
- If last assistant message was a question, treat short replies as conversation
- Pass `chat_ctx` from orchestrator

#### Fix 4: Agent Liveness Heartbeat
- Backend: Publish heartbeat every 5s over LiveKit data channel
- Flutter: Watchdog timer (15s timeout), force disconnect if missed

### What NOT to Do
1. Don't remove intent routing layer — deterministic patterns are valuable fast paths
2. Don't replace router with LLM-only — hybrid approach is correct
3. Don't add more mode categories — current set is correct

### Implementation Order
1. Fix 3 (context-aware router) — highest impact, zero risk
2. Fix 2 (utterance guard) — defense-in-depth
3. Fix 1 (EOU model) — configuration change
4. Fix 4 (heartbeat) — self-contained
