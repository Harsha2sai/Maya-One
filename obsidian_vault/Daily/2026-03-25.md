#### Issue #1: IDENTITY_PATTERNS Regex ✅ FIXED
**File:** `core/context/context_builder.py` lines 16-20 (both directories)

**Problem:** Pattern matched "what is my name" (first-person user identity) and skipped memory retrieval

**Solution:** Updated regex to only match second-person identity questions (MAYA identity)
```python
# OLD: Generic pattern for any identity question
r"\b(what(?:'s| is)\s+your\s+name|who\s+are\s+you)\b"

# NEW: Only for MAYA's identity (second-person)
# DO NOT skip for first-person questions like "what's my name" (user identity = use memory!)
r"\b(what(?:'s| is)\s+your\s+name|who\s+are\s+you|what\s+are\s+you\s+called)\b"
```

**Status:** Applied to both `/phase9_worktree/Agent/` and `/Agent/`
**Impact on test:** No improvement (this layer wasn't the problem for our test case)

#### Issue #2: enforce_budget() Memory Removal ✅ FIXED
**File:** `core/context/token_budget_guard.py` lines 130-195 (both directories)

**Problem:** Function was indiscriminately removing message at index [1] (where memory context sits)
```python
# BEFORE: ❌ Always removes index 1, which is memory location!
while total_tokens > effective_limit and len(current) > 2:
    removed = current.pop(1)
    total_tokens -= estimate_tokens(removed)
```

**Solution:** Intelligent message protection system
```python
# AFTER: ✅ Protects system, memory, and user messages
def is_memory_message(msg: Dict[str, str]) -> bool:
    """Check if this is a memory context message."""
    content = str(msg.get("content", ""))
    return "[Memory]" in content

while total_tokens > effective_limit and len(current) > 2:
    removed_idx = None
    for idx in range(1, len(current) - 1):
        msg = current[idx]
        if msg.get("role") == "system":  # Protect system messages
            continue
        if is_memory_message(msg):  # ✅ NEW: Protect memory messages
            continue
        removed_idx = idx
        break
    
    if removed_idx is None:
        break
    
    removed = current.pop(removed_idx)
    total_tokens -= estimate_tokens(removed)
```

**Protected Messages:**
- System prompts (role == "system")
- Memory contexts (content contains "[Memory]")
- Current user message (last position)

**Droppable Messages:** Old conversation history only

**Status:** Applied to both directories with INFO-level debug logging
**Impact on test:** Memory no longer removed by this layer, but test STILL FAILS (indicates third filter)
