## Action State Reform Plan (Phase 28.5) - COMPLETED TODAY

**Status:** Design Complete - Ready for Implementation  
**Location:** [[Phase_28_5_Action_State_Reform_Plan]]  
**Priority:** CRITICAL - Blocks robust multi-turn tool use

### Problem Identified
The recent "YouTube/Instagram" follow-up failure is not a parsing bug—it's a symptom of the **"Agentic Chasm"**: the gap between single-turn tool execution and conversational state management.

**Key Symptoms:**
- "also Instagram" not understood as app-open context
- "close them" fails to resolve to opened apps
- False "Opened/Closed" confirmations (no verification)
- Three overlapping action paths with no shared state

**Research Insight:** BFCL v4 benchmarks show even top models (GPT-4, Claude) score only **~12%** on agentic memory tasks. This requires architectural reform, not better prompting.

### Solution Designed

**5-Phase Implementation:**
1. **Phase 28.5.1:** Core Schema & Tool Receipts (3-4 days)
   - `ActionIntent`, `ToolReceipt`, `VerificationResult` schemas
   - Structured tool returns with evidence

2. **Phase 28.5.2:** Conversation Action State (4-5 days)
   - `ActionState` class with anaphora resolution
   - "close them" → resolves to opened apps
   - "also X" → recognizes prior action context

3. **Phase 28.5.3:** Verification Layer (4-5 days)
   - Pre/post execution state capture
   - Truthfulness policy (graduated responses)
   - False success detection

4. **Phase 28.5.4:** Unified Routing (5-7 days)
   - Merge FastPathRouter + SystemPlanner into unified `ActionIntent` flow
   - Feature flag for safe rollout

5. **Phase 28.5.5:** Regression Tests (3-4 days)
   - Multi-turn action sequences
   - Anaphora resolution accuracy >90%
   - Verification coverage
