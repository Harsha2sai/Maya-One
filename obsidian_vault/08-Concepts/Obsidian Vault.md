# Obsidian Vault

## Definition
Structured knowledge graph containing atomic notes about the Maya Agent system architecture, components, decisions, bugs, workflows, and concepts.

## Usage in Maya
This vault serves as the single source of truth for:
- System architecture (7-layer runtime chain)
- Component responsibilities and interactions
- Past decisions and their reasoning
- Known bugs and their status
- Development workflows
- Code architecture patterns

## Structure
```
obsidian_vault/
├── 01-Architecture/     # System-wide architecture patterns
├── 02-Components/     # Individual component specifications
├── 03-Memory/       # Memory system components
├── 04-Decisions/    # Design decisions with reasoning
├── 05-Bugs/         # Known bugs and fixes
├── 06-Workflows/    # Development and testing workflows
├── 07-Logs/         # Structured log analysis
├── 08-Concepts/     # Core concepts and definitions
└── Templates/       # Note templates
```
